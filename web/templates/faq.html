FAQ

This page explains what the app does, why it does it this way, and what each analysis step means.

What are we doing?

We estimate password strength with multiple established approaches, then simulate two common attack styles: dictionary matching and brute-force search.

Why separate pages?

Strength scoring, attack simulation, and the explanation are easier to understand when each task has its own page.

How strength scoring works

zxcvbn evaluates real-world patterns, the rule-based layer checks composition cues, and the entropy estimate measures the search space from password length and charset size.

How the final score is built

The tool normalizes each estimator to a common scale and averages them with weights so the final score stays readable from 0 to 100.

How dictionary attacks work here

The app compares the password against the default bundled wordlist, an uploaded file, or both. It can also check case variations and reports the time taken.

How brute force is handled

The app estimates crack time mathematically for normal cases and only tries real brute force for tiny spaces so the tool stays safe.

Wordlist modes

Default
Uses the bundled wordlist that ships with the app.
Upload
Uses only the file you upload for the current run.
Both
Combines the bundled list and your uploaded list.
